# 写作基础

掌握文档写作的基础构建块。本指南涵盖标准Markdown语法、格式选项以及您在每个文档中都会使用的基本元素。 {{ className: 'lead' }}

## Markdown基础

完全支持标准Markdown格式，它构成了所有文档的基础：

**粗体文字**用于强调和重要性  
*斜体文字*用于微妙的强调  
`内联代码`用于技术术语、命令和短代码片段

您可以组合使用：**_粗体和斜体_** 或 **粗体配`代码`**

## 文本格式

### 强调和加粗文本

使用星号或下划线表示强调：

```markdown
_这是斜体文本_
_这也是斜体_

**这是粗体文本**
**这也是粗体**

**_这是粗体和斜体_**
```

### 代码和技术术语

对于内联代码、变量或技术术语，使用反引号：

```markdown
使用 `npm install` 命令安装包。
将 `NODE_ENV` 变量设置为 `production`。
`useState` 钩子管理组件状态。
```

## 列表和组织

### 无序列表

非常适合功能列表、需求或任何非顺序项目：

- 主要功能或要点
- 另一个重要项目
- 第三个考虑因素
  - 嵌套子要点
  - 附加子详情
- 回到主层级

### 有序列表

用于分步说明、程序或优先级项目：

1. 过程中的第一步
2. 包含重要详细信息的第二步
3. 第三步
   1. 带有具体说明的子步骤
   2. 另一个子步骤
4. 最后一步

### 任务列表

非常适合检查清单和进度跟踪：

- [x] 已完成的任务
- [x] 另一个完成的项目
- [ ] 待处理任务
- [ ] 未来增强功能

## 标题和结构

使用适当的标题级别创建清晰的文档层次结构：

```markdown
# 页面标题（H1 - 从元数据自动使用）

## 主要部分（H2）

### 子部分（H3）

#### 详细部分（H4）

##### 次要部分（H5）

###### 最小部分（H6）
```

### 标题最佳实践

- 仅将H1用于页面标题（由元数据处理）
- 主要部分使用H2，子部分使用H3
- 不要跳过标题级别（不要H2 → H4）
- 保持标题描述性和可扫描性
- 使用句子大小写："入门指南"而不是"入门指南"

## 引用和标注

### 块引用

用于重要引用或参考：

> "文档是你写给未来自己的情书。"  
> — Damian Conway

> **重要提示**：这是一个突出显示的引用，包含跨越多行并提供关键信息的附加上下文。

### 多段落引用

> 这是较长引用的第一段。
>
> 这是第二段，继续思考并提供额外的细节和上下文。


## 表格

用于结构化数据的简单表格：

| 功能     | 基础版 | 专业版 | 企业版   |
| -------- | ------ | ------ | -------- |
| 用户数   | 10     | 100    | 无限制   |
| 存储空间 | 1GB    | 10GB   | 100GB    |
| API调用  | 1,000  | 10,000 | 无限制   |
| 支持     | 邮件   | 优先   | 24/7电话 |

### 表格对齐

控制列对齐：

| 左对齐   | 居中对齐 |   右对齐 |
| :------- | :------: | -------: |
| 文本     |   文本   |     文本 |
| 更多内容 | 更多内容 | 更多内容 |

## 特殊字符

使用反斜杠转义特殊Markdown字符：

```markdown
\*不是斜体\*
\`不是代码\`
\# 不是标题
```

## 换行和间距

- 行末使用两个空格进行硬换行
- 使用空行分隔段落
- 在列表中使用行末的`\`进行换行
