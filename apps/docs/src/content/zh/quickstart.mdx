## 选择安装方式

在开始第一次检查之前，您需要选择如何安装 PCU。除了传统的 npm 全局安装，PCU 还支持通过 pnpm、yarn 和 npx 临时运行。以下示例展示了如何安装 PCU：

<CodeGroup>

```bash {{ title: 'npm' }}
# 使用 npm 全局安装 PCU
npm install -g pnpm-catalog-updates
```

```bash {{ title: 'pnpm' }}
# 使用 pnpm 全局安装 PCU
pnpm add -g pnpm-catalog-updates
```

```bash {{ title: 'yarn' }}
# 使用 yarn 全局安装 PCU
yarn global add pnpm-catalog-updates
```

```bash {{ title: 'npx' }}
# 使用 npx 临时运行（无需安装）
npx pnpm-catalog-updates --help
```

</CodeGroup>

## 执行第一次检查

选择安装方式后，您就可以在 pnpm 工作区中执行第一次依赖项检查了。以下示例展示了如何运行 PCU 来检查过时的依赖项，默认显示前10个结果：

<CodeGroup tag="RUN" label="pcu check">

```bash {{ title: 'npm' }}
# 检查过时的依赖项
pcu check --outdated
```

```bash {{ title: 'pnpm' }}
# 检查特定包的更新
pcu check react --latest
```

```bash {{ title: 'yarn' }}
# 检查整个工作区
pcu check --workspace --verbose
```

```bash {{ title: 'npx' }}
# 更新 catalog 配置
npx pnpm-catalog-updates update --interactive
```

</CodeGroup>
