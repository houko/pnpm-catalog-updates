# ライティング基礎

ドキュメントライティングの基本的な構成要素をマスターしましょう。このガイドでは、標準のMarkdown構文、書式設定オプション、およびすべてのドキュメントで使用する基本要素について説明します。 {{ className: 'lead' }}

## Markdown基礎

すべてのドキュメントの基盤となる標準的なMarkdown形式を完全サポート：

**太字テキスト**は強調と重要性のため  
*斜体テキスト*は微妙な強調のため  
`インラインコード`は技術用語、コマンド、短いコードスニペットのため

組み合わせて使用できます：**_太字と斜体_** または **太字と`コード`**

## テキスト形式

### 強調と太字テキスト

アスタリスクまたはアンダースコアを使用して強調を表現：

```markdown
_これは斜体テキストです_
_これも斜体です_

**これは太字テキストです**
**これも太字です**

**_これは太字と斜体です_**
```

### コードと技術用語

インラインコード、変数、または技術用語にはバッククォートを使用：

```markdown
`npm install` コマンドを使用してパッケージをインストールします。
`NODE_ENV` 変数を `production` に設定します。
`useState` フックはコンポーネントの状態を管理します。
```

## リストと整理

### 順序なしリスト

機能リスト、要件、またはあらゆる非順序項目に最適：

- 主要機能または要点
- 別の重要な項目
- 第3の考慮事項
  - ネストされたサブポイント
  - 追加のサブ詳細
- メインレベルに戻る

### 順序ありリスト

ステップバイステップの指示、手順、または優先順位付きの項目に使用：

1. プロセスの最初のステップ
2. 重要な詳細を含む第2ステップ
3. 第3ステップ
   1. 具体的な指示を含むサブステップ
   2. 別のサブステップ
4. 最後のステップ

### タスクリスト

チェックリストと進捗追跡に最適：

- [x] 完了したタスク
- [x] 別の完了した項目
- [ ] 保留中のタスク
- [ ] 将来の機能強化

## リンクとナビゲーション

### 内部リンク

ドキュメント内の他のページにリンク：

```markdown
セットアップ手順については[クイックスタートガイド](/quickstart)をご覧ください。
完全なエンドポイント詳細については[APIリファレンス](/api)をご覧ください。
```

例：

- [コマンドリファレンスガイド](/command-reference)
- [トラブルシューティング](/troubleshooting)
- [SDKドキュメント](/sdks)

### 外部リンク

外部リソースにリンク：

```markdown
ソースコードについては[GitHub](https://github.com)をご覧ください。
[公式ドキュメント](https://example.com/docs)をお読みください。
```

### アンカーリンク

ページ内の特定の部分にリンク：

```markdown
上記の[テキスト形式](#テキスト形式)セクションにジャンプします。
下記の[ベストプラクティス](#ベストプラクティス)をご覧ください。
```

## 見出しと構造

適切な見出しレベルを使用して明確なドキュメント階層を作成：

```markdown
# ページタイトル（H1 - メタデータから自動使用）

## メインセクション（H2）

### サブセクション（H3）

#### 詳細セクション（H4）

##### マイナーセクション（H5）

###### 最小セクション（H6）
```

### 見出しのベストプラクティス

- H1はページタイトルのみに使用（メタデータで処理）
- メインセクションにはH2、サブセクションにはH3を使用
- 見出しレベルをスキップしない（H2 → H4は禁止）
- 見出しを説明的でスキャン可能に保つ
- 文の大文字小文字を使用：「スタートガイド」ではなく「スタートガイド」

## 引用とコールアウト

### ブロック引用

重要な引用や参考に使用：

> "ドキュメントは未来の自分への愛の手紙です。"  
> — Damian Conway

> **重要な注意**：これは重要な情報を複数行にわたって提供し、追加のコンテキストを含む強調表示された引用です。

### 複数段落の引用

> これは長い引用の最初の段落です。
>
> これは2番目の段落で、思考を続け、追加の詳細とコンテキストを提供します。

## 水平区切り線

水平区切り線を使用してメインセクションを分離：

```markdown
---
```

視覚的な分離を作成：

## テーブル

構造化データのためのシンプルなテーブル：

| 機能        | ベーシック | プロ   | エンタープライズ |
| ----------- | ---------- | ------ | ---------------- |
| ユーザー数  | 10         | 100    | 無制限           |
| ストレージ  | 1GB        | 10GB   | 100GB            |
| API呼び出し | 1,000      | 10,000 | 無制限           |
| サポート    | メール     | 優先   | 24/7電話         |

### テーブルの配置

列の配置を制御：

| 左配置       |   中央配置   |       右配置 |
| :----------- | :----------: | -----------: |
| テキスト     |   テキスト   |     テキスト |
| その他の内容 | その他の内容 | その他の内容 |

## 特殊文字

バックスラッシュを使用して特殊なMarkdown文字をエスケープ：

```markdown
\*斜体ではない\*
\`コードではない\`
\# 見出しではない
```

## 改行とスペース

- 行末で2つのスペースを使用してハード改行
- 空行を使用して段落を分離
- リスト内で行末の`\`を使用して改行
